- 부모 디렉터리가 `private_paths`로 미게시인 상태에서 하위 파일/디렉터리 하나만 게시할 때는 부모 규칙을 제거하지 않고 `public_paths` 공개 예외를 추가한다.
- `public_paths` 공개 예외가 있는 하위 항목까지 접근할 수 있도록 부모 디렉터리는 탐색용 컨테이너로 표시할 수 있다.
- 하위 항목이 `public_paths`로 게시된 상위 디렉터리는 게시 상태로 표시하고, 상위 디렉터리를 미게시로 바꾸면 하위 공개 예외를 함께 제거한다.
- 루트의 앱 구동/관리용 `js`, `html`, 작업용 `md` 파일은 파일 탐색/검색 데이터에서 제외하고, 콘텐츠성 문서는 유지한다.
- 퍼블리싱 패널은 GitHub 원격 `grant.json`의 SHA를 기준으로 서버 동기화 상태를 확인하고, 원격 변경이 있으면 자동 덮어쓰기 대신 노티와 수동 불러오기를 제공한다.
- `GitHub에 반영` 저장 성공 후에는 저장 커밋 SHA 이후 원격 `main` HEAD가 전진하는지 짧게 폴링해 Actions 생성 커밋을 확인하고 최신 정보를 자동으로 다시 불러온다.
- 퍼블리싱 패널의 동기화 표시는 `게시 설정(grant.json)`과 `인덱스(files.json)`를 분리하고, 원격 `files.json`과 로컬 `files.json`이 다르면 `git pull 필요`를 표시한다.
- 메인 탐색기 스플릿터는 목록 폭만 고정하고 미리보기 컬럼은 남은 폭을 채우게 유지한다.
- 게시 상태 조건 필터는 로컬 점검 환경(`localhost`, `127.0.0.1`, `file:`)에서만 표시/적용하고, `PublishingUtils.matchesPublishingFilter`를 기준으로 메인/자료실/워드클라우드가 같은 공개 예외 규칙을 사용한다.
